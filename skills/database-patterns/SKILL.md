# Database Patterns

## General Conventions

- ORM: SQLAlchemy (async, using `asyncpg` driver).
- Migrations: Alembic — every schema change ships with a migration file. Never alter tables manually.
- All tables have `id` (UUID primary key), `created_at`, and `updated_at` columns.
- UUIDs generated by the application layer (`uuid.uuid4()`), not the database.
- Timestamps stored as `TIMESTAMPTZ` (UTC). Never use `TIMESTAMP WITHOUT TIME ZONE`.
- Soft deletes preferred over hard deletes — use `is_active BOOLEAN DEFAULT TRUE`.
- Snake_case for all column and table names.
- Monetary values stored as `INTEGER` (cents). Include a `currency` column (`VARCHAR(3)`, e.g. `"CZK"`, `"EUR"`).

## Base Model

```python
import uuid
from datetime import datetime
from sqlalchemy.orm import DeclarativeBase, mapped_column, Mapped
from sqlalchemy import DateTime, func

class Base(DeclarativeBase):
    pass

class TimestampMixin:
    id: Mapped[uuid.UUID] = mapped_column(primary_key=True, default=uuid.uuid4)
    created_at: Mapped[datetime] = mapped_column(DateTime(timezone=True), server_default=func.now())
    updated_at: Mapped[datetime] = mapped_column(DateTime(timezone=True), server_default=func.now(), onupdate=func.now())
```

## Table: `users`

Stores customer and admin accounts.

```sql
CREATE TABLE users (
    id            UUID PRIMARY KEY,
    email         VARCHAR(255) NOT NULL UNIQUE,
    password_hash VARCHAR(255) NOT NULL,
    full_name     VARCHAR(255),
    role          VARCHAR(20)  NOT NULL DEFAULT 'customer',  -- 'customer' | 'admin'
    is_active     BOOLEAN      NOT NULL DEFAULT TRUE,
    created_at    TIMESTAMPTZ  NOT NULL DEFAULT now(),
    updated_at    TIMESTAMPTZ  NOT NULL DEFAULT now()
);
```

- `role` is a plain `VARCHAR` with an application-level check, not a PostgreSQL enum (easier to extend).
- No OAuth columns in MVP — email/password only.

## Table: `products`

Catalog figurines available for immediate order.

```sql
CREATE TABLE products (
    id              UUID         PRIMARY KEY,
    slug            VARCHAR(120) NOT NULL UNIQUE,   -- URL-safe identifier, e.g. "steampunk-dragon-medium"
    name            VARCHAR(255) NOT NULL,
    description     TEXT,
    category        VARCHAR(80)  NOT NULL,           -- 'fantasy' | 'meme' | 'seasonal' | 'animals' | 'scifi' | ...
    size_tier       VARCHAR(10)  NOT NULL,           -- 'small' | 'medium' | 'large'
    price_cents     INTEGER      NOT NULL,
    currency        VARCHAR(3)   NOT NULL DEFAULT 'CZK',
    model_file_url  TEXT         NOT NULL,           -- S3/GCS path to the print-ready .stl/.obj
    preview_url     TEXT,                            -- rendered image for storefront
    is_active       BOOLEAN      NOT NULL DEFAULT TRUE,
    created_at      TIMESTAMPTZ  NOT NULL DEFAULT now(),
    updated_at      TIMESTAMPTZ  NOT NULL DEFAULT now()
);

CREATE INDEX idx_products_category    ON products(category) WHERE is_active = TRUE;
CREATE INDEX idx_products_size_tier   ON products(size_tier) WHERE is_active = TRUE;
```

Size tier semantics:

| Value    | Approx. height | Price tier |
|----------|---------------|------------|
| `small`  | ~8 cm         | lowest     |
| `medium` | ~15 cm        | mid        |
| `large`  | ~25 cm        | premium    |

## Table: `orders`

Tracks the full lifecycle of each purchase.

```sql
CREATE TABLE orders (
    id                  UUID         PRIMARY KEY,
    user_id             UUID         NOT NULL REFERENCES users(id),
    status              VARCHAR(20)  NOT NULL DEFAULT 'placed',
    -- State machine: placed → processing → printing → shipped → delivered
    -- Cancellation: placed → cancelled (only valid from 'placed')

    -- Line items are stored in order_items (see below)
    total_cents         INTEGER      NOT NULL,
    currency            VARCHAR(3)   NOT NULL DEFAULT 'CZK',

    stripe_session_id   VARCHAR(255),               -- Checkout Session ID
    stripe_payment_id   VARCHAR(255),               -- PaymentIntent ID

    shipping_address    JSONB        NOT NULL,       -- snapshot at order time
    tracking_number     VARCHAR(100),
    notes               TEXT,

    created_at          TIMESTAMPTZ  NOT NULL DEFAULT now(),
    updated_at          TIMESTAMPTZ  NOT NULL DEFAULT now()
);

CREATE INDEX idx_orders_user_id ON orders(user_id);
CREATE INDEX idx_orders_status  ON orders(status);
```

### Order State Machine

```
placed ──► processing ──► printing ──► shipped ──► delivered
  │
  └──► cancelled   (only from 'placed')
```

- `placed`: payment captured by Stripe, awaiting internal processing.
- `processing`: print file verified and sent to MCAE.
- `printing`: MCAE confirmed they are printing.
- `shipped`: tracking number received.
- `delivered`: confirmed by courier or customer.
- `cancelled`: customer-requested cancellation, only valid from `placed`.

State transitions are enforced in the service layer, not the database. Invalid transitions raise `HTTP 409`.

### Table: `order_items`

```sql
CREATE TABLE order_items (
    id              UUID        PRIMARY KEY,
    order_id        UUID        NOT NULL REFERENCES orders(id) ON DELETE CASCADE,
    product_id      UUID        REFERENCES products(id),           -- NULL for AI-custom items
    generation_job_id UUID      REFERENCES generation_jobs(id),   -- NULL for catalog items
    quantity        INTEGER     NOT NULL DEFAULT 1,
    unit_price_cents INTEGER    NOT NULL,
    currency        VARCHAR(3)  NOT NULL DEFAULT 'CZK',
    size_tier       VARCHAR(10) NOT NULL,
    snapshot_name   VARCHAR(255) NOT NULL,  -- product name at time of order
    created_at      TIMESTAMPTZ NOT NULL DEFAULT now(),
    updated_at      TIMESTAMPTZ NOT NULL DEFAULT now(),
    CHECK (product_id IS NOT NULL OR generation_job_id IS NOT NULL)
);
```

## Table: `generation_jobs`

AI "Prompt to Print" pipeline state.

```sql
CREATE TABLE generation_jobs (
    id                  UUID         PRIMARY KEY,
    user_id             UUID         NOT NULL REFERENCES users(id),
    prompt              TEXT         NOT NULL,
    size_tier           VARCHAR(10)  NOT NULL,   -- 'small' | 'medium' | 'large'

    status              VARCHAR(30)  NOT NULL DEFAULT 'queued',
    -- queued → generating → awaiting_review → approved → rejected → failed
    -- After approved → order is created and job links to it

    ai_provider         VARCHAR(50),             -- 'meshy' | 'tripo3d' | null
    ai_job_id           VARCHAR(255),            -- external provider job ID
    model_file_url      TEXT,                    -- raw AI output file
    repaired_file_url   TEXT,                    -- after automated mesh repair
    preview_url         TEXT,                    -- render sent to customer

    deposit_cents       INTEGER,                 -- 50% deposit amount
    remaining_cents     INTEGER,                 -- 50% charged on approval
    currency            VARCHAR(3) NOT NULL DEFAULT 'CZK',
    stripe_deposit_session_id   VARCHAR(255),
    stripe_remaining_session_id VARCHAR(255),

    rejection_reason    TEXT,                    -- customer-supplied on rejection
    technician_notes    TEXT,                    -- internal QA notes

    order_id            UUID REFERENCES orders(id),   -- set once approved and order created

    created_at          TIMESTAMPTZ NOT NULL DEFAULT now(),
    updated_at          TIMESTAMPTZ NOT NULL DEFAULT now()
);

CREATE INDEX idx_generation_jobs_user_id ON generation_jobs(user_id);
CREATE INDEX idx_generation_jobs_status  ON generation_jobs(status);
```

### Generation Job State Machine

```
queued → generating → awaiting_review → approved → (order created)
                              │
                              └──► rejected → queued   (customer can retry)
generating → failed                                    (AI provider error)
```

- `queued`: job accepted, waiting for AI worker to pick it up.
- `generating`: AI provider call is in-flight.
- `awaiting_review`: model generated, render ready, waiting for customer approval.
- `approved`: customer approved preview — triggers remaining payment + order creation.
- `rejected`: customer rejected — job can be re-queued with the same or revised prompt.
- `failed`: unrecoverable AI or mesh-repair error — customer notified, deposit refunded.

## Migrations

- Use Alembic with `--autogenerate` as a starting point, but always review the generated diff before committing.
- Migration file naming: Alembic default (timestamp + slug) — do not rename.
- Each migration must be reversible (`downgrade` function must be implemented).
- Never run `alembic upgrade head` in CI against production directly — use a separate migration job in the deploy pipeline.

## Query Patterns

- Always use `SELECT ... WHERE is_active = TRUE` when listing products.
- Use `RETURNING id` on inserts to avoid a follow-up SELECT.
- Avoid N+1: use `selectinload` or `joinedload` for related collections in SQLAlchemy.
- Paginate with `LIMIT`/`OFFSET` at the DB layer, not in Python.

```python
# Correct pagination pattern
result = await db.execute(
    select(Product)
    .where(Product.is_active == True)
    .order_by(Product.created_at.desc())
    .limit(page_size)
    .offset((page - 1) * page_size)
)
```
