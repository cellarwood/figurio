# Database Patterns

## Purpose

Define PostgreSQL schema conventions, migration rules, and query patterns for the Figurio platform. All database changes must follow these standards.

## Naming Conventions

- **Tables**: snake_case, plural (e.g., `products`, `order_items`).
- **Columns**: snake_case (e.g., `created_at`, `size_tier`).
- **Primary keys**: `id` column, UUID type, generated by application (not database serial).
- **Foreign keys**: `{referenced_table_singular}_id` (e.g., `customer_id`, `product_id`).
- **Indexes**: `ix_{table}_{column(s)}` (e.g., `ix_products_category`).
- **Constraints**: `ck_{table}_{description}` for checks, `uq_{table}_{column}` for unique.
- **Enum types**: `{domain}_status` or `{domain}_type` (e.g., `order_status`, `size_tier`).

## Core Tables

### products

```sql
CREATE TABLE products (
    id UUID PRIMARY KEY,
    sku VARCHAR(50) UNIQUE NOT NULL,        -- FIG-GAME-001
    name VARCHAR(255) NOT NULL,
    description TEXT,
    category VARCHAR(100) NOT NULL,         -- gaming, anime, pets, custom
    base_price_czk INTEGER NOT NULL,        -- price in hellers (smallest unit)
    base_price_eur INTEGER,                 -- price in cents (for EU expansion)
    model_file_url VARCHAR(500),            -- STL file path in object storage
    preview_file_url VARCHAR(500),          -- GLB file path for 3D viewer
    thumbnail_url VARCHAR(500),
    is_active BOOLEAN DEFAULT TRUE,         -- soft delete
    is_custom BOOLEAN DEFAULT FALSE,
    created_at TIMESTAMPTZ DEFAULT NOW(),
    updated_at TIMESTAMPTZ DEFAULT NOW()
);
```

### product_size_tiers

```sql
CREATE TABLE product_size_tiers (
    id UUID PRIMARY KEY,
    product_id UUID NOT NULL REFERENCES products(id) ON DELETE CASCADE,
    size_tier size_tier NOT NULL,            -- S, M, L, XL
    height_cm DECIMAL(5,1) NOT NULL,
    price_czk INTEGER NOT NULL,
    price_eur INTEGER,
    weight_grams INTEGER,
    UNIQUE(product_id, size_tier)
);
```

### customers

```sql
CREATE TABLE customers (
    id UUID PRIMARY KEY,
    email VARCHAR(255) UNIQUE NOT NULL,
    name VARCHAR(255) NOT NULL,
    phone VARCHAR(50),
    stripe_customer_id VARCHAR(255) UNIQUE,
    default_shipping_address_id UUID,
    created_at TIMESTAMPTZ DEFAULT NOW(),
    updated_at TIMESTAMPTZ DEFAULT NOW()
);
```

### orders

```sql
CREATE TABLE orders (
    id UUID PRIMARY KEY,
    order_number VARCHAR(20) UNIQUE NOT NULL, -- FIG-20260408-0001
    customer_id UUID NOT NULL REFERENCES customers(id),
    status order_status NOT NULL DEFAULT 'pending',
    subtotal_czk INTEGER NOT NULL,
    shipping_czk INTEGER NOT NULL,
    total_czk INTEGER NOT NULL,
    shipping_provider VARCHAR(50),           -- zasilkovna, dhl
    tracking_number VARCHAR(100),
    shipping_address_id UUID REFERENCES shipping_addresses(id),
    notes TEXT,
    created_at TIMESTAMPTZ DEFAULT NOW(),
    updated_at TIMESTAMPTZ DEFAULT NOW()
);
```

### order_items

```sql
CREATE TABLE order_items (
    id UUID PRIMARY KEY,
    order_id UUID NOT NULL REFERENCES orders(id) ON DELETE CASCADE,
    product_id UUID NOT NULL REFERENCES products(id),
    size_tier size_tier NOT NULL,
    quantity INTEGER NOT NULL DEFAULT 1,
    unit_price_czk INTEGER NOT NULL,
    custom_order_id UUID REFERENCES custom_orders(id)  -- NULL for catalog items
);
```

### payments

```sql
CREATE TABLE payments (
    id UUID PRIMARY KEY,
    order_id UUID NOT NULL REFERENCES orders(id),
    stripe_payment_intent_id VARCHAR(255) UNIQUE NOT NULL,
    amount_czk INTEGER NOT NULL,
    currency VARCHAR(3) DEFAULT 'czk',
    status payment_status NOT NULL DEFAULT 'pending',
    paid_at TIMESTAMPTZ,
    refunded_at TIMESTAMPTZ,
    created_at TIMESTAMPTZ DEFAULT NOW()
);
```

### custom_orders

```sql
CREATE TABLE custom_orders (
    id UUID PRIMARY KEY,
    customer_id UUID NOT NULL REFERENCES customers(id),
    prompt TEXT NOT NULL,
    status custom_order_status NOT NULL DEFAULT 'submitted',
    revision_count INTEGER DEFAULT 0,
    max_revisions INTEGER DEFAULT 3,
    generated_model_url VARCHAR(500),
    repaired_model_url VARCHAR(500),
    preview_glb_url VARCHAR(500),
    preview_thumbnail_url VARCHAR(500),
    generation_service VARCHAR(100),         -- meshy, tripo3d, etc.
    generation_cost_usd DECIMAL(10,4),
    created_at TIMESTAMPTZ DEFAULT NOW(),
    updated_at TIMESTAMPTZ DEFAULT NOW()
);
```

### custom_order_previews

```sql
CREATE TABLE custom_order_previews (
    id UUID PRIMARY KEY,

    'printing', 'shipped', 'delivered', 'cancelled'

```

## Migration Rules

- **Tool**: Alembic with auto-generation from SQLAlchemy models.
- **Always reversible**: Every migration must have a working `downgrade()`.
- **One concern per migration**: Do not mix schema changes with data migrations.
- **Naming**: `{timestamp}_{description}.py` (e.g., `20260408_add_custom_orders_table.py`).
- **Test downgrades**: Run `alembic downgrade -1` then `alembic upgrade head` in CI.
- **No dropping columns in prod** without a deprecation period (mark nullable first, remove later).

## Index Strategy

```sql
-- Product lookups
CREATE INDEX ix_products_category ON products(category) WHERE is_active = TRUE;
CREATE INDEX ix_products_sku ON products(sku);

-- Order queries
CREATE INDEX ix_orders_customer_id ON orders(customer_id);
CREATE INDEX ix_orders_status ON orders(status);
CREATE INDEX ix_orders_created_at ON orders(created_at DESC);

-- Custom order pipeline
CREATE INDEX ix_custom_orders_status ON custom_orders(status);
CREATE INDEX ix_custom_orders_customer_id ON custom_orders(customer_id);

-- Payment reconciliation
CREATE INDEX ix_payments_stripe_payment_intent_id ON payments(stripe_payment_intent_id);
CREATE INDEX ix_payments_order_id ON payments(order_id);
```

## Soft Deletes

Products use soft deletes via `is_active` flag:
- `DELETE /api/v1/products/{sku}` sets `is_active = FALSE`.
- All product queries filter `WHERE is_active = TRUE` by default.
- Admin queries can include inactive products with `?include_inactive=true`.
- Reason: orders reference products; hard deletes would break order history.

## Query Patterns

- Use SQLAlchemy ORM for standard CRUD operations.
- Use SQLAlchemy Core for complex reports and aggregations.
- No raw SQL strings in application code.
- Always use parameterized queries (SQLAlchemy handles this automatically).
- Use `select_in_loading` for eager loading related objects to avoid N+1 queries.
