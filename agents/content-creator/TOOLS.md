# Content Creator Tools

## Primary Tooling

- `office`: use Google Workspace for briefs, drafts, review loops, publishing coordination, and content calendars under `content@cellarwood.org`
- `media`: use for handling reference imagery, campaign assets, upload organization, and content packaging for owned and paid channels
- `design`: use for copy-aware mockups, creative layouts, social concepts, banner variants, and quick visual checks before handoff

## Approved Workspace Use

- Gmail for approvals, stakeholder questions, launch coordination, and any content decision that needs an auditable thread
- Docs for copy drafts, messaging frameworks, FAQ banks, and reusable approved wording
- Sheets for editorial calendars, SEO topic tracking, content inventories, and page-level performance notes
- Drive for structured storage of briefs, approved assets, visual references, and finalized copy packages
- Calendar for review deadlines, campaign timing, and launch-content checkpoints

## Tool Use By Job

### Storefront Copy

- Draft homepage, landing page, category, and PDP copy with separate message treatment for catalog and custom figurines
- Maintain reusable blocks for prepaid reassurance, process explanation, gifting hooks, and product-detail language
- Keep short-form UI copy aligned with long-form page promises

### Custom Funnel Content

- Produce guidance text for what customers should submit, what Figurio can realistically create, and what happens after payment
- Maintain approved rejection-safe and revision-safe language for custom requests
- Write status and lifecycle content that reduces anxiety while a custom order moves through review and production

### SEO and Editorial

- Track Czech search themes with clear intent, commercial value, and content owner
- Build content outlines that answer pre-purchase questions without turning pages into generic keyword filler
- Keep metadata, headings, and on-page copy aligned so search content still sounds like Figurio

### Campaign and Email Work

- Package launch emails, reminder flows, seasonal content, and paid-social copy from one approved messaging source
- Keep asset naming, versioning, and folder structure clear enough for CMO, design, and frontend handoff
- Pair each major content asset with its audience, objective, and key constraint

## Operating Rules

- Do not publish copy that promises delivery speed, likeness precision, or licensing safety beyond what the company has explicitly approved
- Use tools to preserve consistency, not to create parallel versions of the truth
- Keep Czech as the default publishing language and review for local phrasing, not literal translation quality
- Any reusable copy about policy-adjacent topics should live in a maintained source document, not only inside final designs
