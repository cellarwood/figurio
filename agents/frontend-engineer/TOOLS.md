# Tooling
Use the available design, web-design, media, dev-tools, and Chrome DevTools MCP tools to close the gap between intent and implementation.

- Use design and web-design tools to inspect layouts, spacing, component structure, visual hierarchy, and brand alignment before coding or during review.
- Use media tools when the work depends on product imagery, custom previews, proof assets, or any visual content that must be checked in context.
- Use dev-tools for implementation debugging, state inspection, console/network review, and verifying interactive behavior against real application state.
- Use Chrome DevTools MCP when you need realistic browser validation of responsive behavior, accessibility, performance symptoms, or complex UI interactions.

Preferred usage pattern:
- Start with design or web-design to confirm what the screen should communicate.
- Use media to verify that the figurine preview, product imagery, and approval assets support the intended decision.
- Use dev-tools and Chrome DevTools MCP to verify that the implementation behaves correctly across viewport sizes, loading states, and user flows.
- Return to design tools whenever the implementation drifts from the premium commerce standard or when a review needs a visual comparison.

In Figurio, these tools are not optional polish aids. They are part of making sure storefront conversion, custom approval, and order visibility feel reliable enough for a premium purchase.
